# Desktop 客户端 — 支持与 SLA（草案）

> 依据 ADR 0006 §9：桌面与云端 **独立版本线、独立 SLA**。  
> 本文档为运营/支持口径草案，数字与响应时效可后续修订。

## 产品边界

| 项目 | 桌面 | 云端 SaaS |
|------|------|-----------|
| 主要交付物 | 可安装客户端（Tauri 等） | 托管 API + Web |
| 版本线 | 独立版本号与 changelog | 独立版本号与 changelog |
| LLM 用量 | **本地 / BYOK 不上报、不计入云配额** | User rolling 配额 |
| 连云功能 | 可选；走云时适用 **云端 SLA** | 本 SLA 不适用 |

## 版本线

- 桌面版本号与云 API **解耦**（例如 Desktop 1.x 可对应最低云 API `v1`）。
- 发布说明分列：`docs/` 下 Cloud vs Desktop release notes。
- 破坏性变更：桌面 changelog 注明「最低兼容云 API 版本」（若该版本含连云能力）。

## 支持范围（建议口径）

### 我们负责

- 安装包完整性、许可激活/校验缺陷  
- 桌面 UI/本地功能崩溃、明确的产品缺陷  
- 文档中承诺的连云能力在 **云端可用** 时的客户端联调问题  

### 默认不在本 SLA 内（用户侧）

- BYOK API Key 错误、厂商限流、余额不足  
- 用户本机模型进程/GPU/环境配置  
- 用户修改系统导致的客户端无法启动（可 best-effort 协助）  
- **本机进程「可用性 99.x%」** — 不承诺

## 与云端 SLA 的关系

- 用户登录云并调用云 API 时：**服务端可用性与数据** 以 **云端 SLA** 为准。  
- 桌面仅保证：在云端正常时，客户端能按文档完成认证与调用编排。
