# Handoff: ADR-0006 Stage 1 — ReActLoop 骨架 + Chat 模式

> **⛔ 归档（2026-06-08）** — 含过时 `native_tools` / disclosure 描述。以 [ADR-0007](docs/adr/0007-react-phased-context-disclosure.md) 为准。  
> 索引：[docs/agents/ARCHIVE-superseded-by-adr-0007.md](docs/agents/ARCHIVE-superseded-by-adr-0007.md)

## 上下文

ADR-0006-revised.md 已定稿（`docs/adr/0006-unified-agent-loop-revised.md`）。
核心决策：统一 ReActLoop（Reason→Act→Observe）替代 v5 三个独立 Strategy state machine。

关键设计：
- 每模式 0-1 个原生 tool（RAG: dense, Search: web_search, Chat: 空）
- 其他能力走 Skill 渐进披露 + SDK（codegen Python 沙箱）
- ReAct 循环内部不流式，循环结束后进入独立 Synthesis Phase 流式输出
- Free tier 最低 budget = 2 轮（支持 action + synthesis）
- 累积式 Skill 披露（已加载的 Skill 保留在 system prompt 中）
- Auto 披露用 rule-based/regex 匹配（非语义向量）
- 沙箱连续 2 次报错触发降级/兜底

### Step 2: 新建 ReActLoop 核心文件

文件：`crates/app/src/agents/loop/mod.rs`
```rust
pub struct ReActLoop { ... }
pub struct ReActIterationRecord { ... }

impl ReActLoop {
    pub async fn run(
        &self,
        mode: &ModeConfig,
        request: AgentRequest,
        sink: &dyn AgentEventSink,
    ) -> Result<AgentRunResult, AppError>;
}
```
循环逻辑：
1. 渐进披露 → 选择本轮新增 Skills（累积到 disclosed_skills）
2. 构造 system prompt = base + 累积 skills 描述
3. 调 LLM（complete_with_tools，只带 mode.native_tools）
4. 处理输出：原生 tool_call / `<code>` 标签 / 直接 content
5. 执行 → observation 回环
6. 终止条件：budget / cancellation / 沙箱错误 / 直接 content
7. 进入 Synthesis Phase（独立流式 LLM 调用生成最终答案）

### Step 5: 新建 Skill 渐进披露

文件：`crates/app/src/agents/loop/skills.rs`
```rust

impl SkillDisclosure {
    pub fn progressive_disclose(

- `Index`：返回 skill 索引（id + 一句话描述）

你是 Context OS 的对话助手。你帮助用户解答问题、进行讨论、提供建议。
</context>

<instructions>
1. 直接回答用户问题，保持简洁和友好
2. 如果需要计算或查询信息，在回复中输出 <code language="python"> 标签包裹的 Python 代码

- 不确定时坦诚告知用户
</constraints>
```

## 关键约束

1. **Chat 模式 native_tools = []**：LLM 看不到任何 tool schema
2. **Free tier budget >= 2**：react_loop.rs 中 search/chat Free = 2
3. **Synthesis Phase 必须流式**：使用 `llm.complete_stream()` 或等效 API
4. **`<code>` 标签解析需容错**：支持 markdown code block fallback
5. **Skill 累积不重复**：同一 skill ID 只加载一次
6. **不删除旧 ChatStrategy**：标记 `#[deprecated]`，保留并行
7. **不改前端 SSE 协议**：复用现有 `AgentEvent` 变体
