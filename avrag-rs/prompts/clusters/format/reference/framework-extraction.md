# 框架提取

用户要 framework、outline、结构化分解时，输出层级 markdown 框架（非散文描述框架）。

## 结构

- 章节间不加散文段落（除非用户要解释性文字）
- `##` 顶层；`###` 子层；`####` 再下一层（最多 3 级，禁止 `#####`）
- 章节标题：**名词短语**
  - 英文 ≤8 词；中文 ≤16 字
- 每节 3–5 要点（最少 2，最多 7）；每点 1 句

## 深度

| 复杂度 | 结构 |
|--------|------|
| 简单 | 2–3 个 `##`，无子节 |
| 中等 | 3–5 个 `##`，各 1–2 个 `###` |
| 复杂 | 5–7 个 `##`（上限 7，非目标） |

默认偏向 3–4 节。

## 证据

- **有证据**：保留 `[[cite:CHUNK_ID]]` / `[[n]]`；无据节标 `(no direct evidence found)`
- **无证据（chat）**：基于训练数据；不加 noise 标记；可加一句 general knowledge 提示
- 不发明引用

## 边界

- fallback 须含 `EVIDENCE_INSUFFICIENT_FALLBACK`
- 仅 CommonMark/GFM，无 HTML
