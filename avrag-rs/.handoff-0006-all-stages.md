### 3.3 关键实现要求

**ReActLoop::run 流程**:
1. 加载 mode config（YAML）
2. 初始化 `messages` = `[system prompt]` + `user_query`
3. Phase 1 ReAct 循环：
   - 渐进披露 → 累积 skills
   - 构造 system prompt = base + 累积 skills
   - 调 `complete_with_tools(messages, native_tools)`
   - 解析输出：tool_call / `<code>` / content
   - 执行 → observation 回环
   - 终止条件：budget / cancellation / content / 连续沙箱错误>=2
4. Phase 2 Synthesis：
   - 构造 synthesis prompt（无 tool）
   - 调 `complete_stream()` 流式输出到 sink
   - 返回最终答案

**Chat 模式特殊约束**:
- `native_tools = []`
- LLM 通过 `<code>` 调 SDK（如 `client.calculate()`）
- 无兜底路径——LLM 不输出代码 = 直接回答
- budget = 2（Free/Pro 统一，实际 YAML 中配置）

### 4.3 关键实现要求

**RAG 原生 tool**: `dense_retrieval`
```json
{
  "name": "dense_retrieval",
  "parameters": {
    "type": "object",
    "properties": {
      "query": { "type": "string" },
      "top_k": { "type": "integer", "default": 10, "minimum": 1, "maximum": 50 }
    },
    "required": ["query"]
  }
}
```

**SDK 封装**（Python 沙箱内可用）:
```python
class Client:
    async def dense_search(self, query: str, top_k: int = 10, method: str = "auto") -> list[Chunk]
    async def lexical_search(self, query: str, top_k: int = 10) -> list[Chunk]
    async def graph_search(self, query: str, depth: int = 2) -> list[Chunk]
    async def rerank(self, query: str, chunks: list[Chunk], top_n: int = 5) -> list[Chunk]
    async def chunk_fetch(self, chunk_id: str) -> Chunk
    async def doc_summary(self, doc_ids: list[str], level: str = "doc") -> list[DocSummary]
    async def recall(self, tags: list[str] | None = None, limit: int = 20) -> list[Message]
    async def remember(self, operations: list[dict]) -> dict
```

**RAG 兜底路径**:
- 触发条件：ReAct 循环异常退出（budget 耗尽 / 连续沙箱错误>=2 / 无有效输出）
- 行为：自动用 `request.user_query` 调 `dense_retrieval(top_k=10)`
- 结果注入 messages 作为 observation，进入 Synthesis Phase

**dense 三重路径**:
1. 路径 A（主）：Skill 引导 codegen → `client.dense_search(...)`
2. 路径 B（简单）：原生 tool_call(dense_retrieval)
3. 路径 C（兜底）：系统自动 dense 检索

### 7.2 `<code>` 标签解析器

**文件**: `crates/app/src/agents/loop/parse.rs`

```rust
pub fn parse_llm_output(content: &str) -> LlmOutput {
    // 1. 检查原生 tool_calls（OpenAI 协议）
    if let Some(tool_calls) = parse_native_tool_calls(content) {
        return LlmOutput::NativeToolCalls(tool_calls);
    }
    
    // 2. 检查 <code> 标签
    if let Some(code) = extract_code_block(content) {
        return LlmOutput::CodeBlock(code);
    }
    
    // 3. 检查 markdown code block fallback
    if let Some(code) = extract_markdown_code_block(content) {
        return LlmOutput::CodeBlock(code);
    }
    
    // 4. 纯 content
    LlmOutput::Content(content.to_string())
}
```

## 九、风险提醒

| 风险 | 阶段 | 缓解 |
|---|---|---|
| 沙箱性能瓶颈 | 1-3 | 预热沙箱；高频操作鼓励原生 tool_call |
| `<code>` 解析失败 | 1-3 | 容错解析（markdown fallback）；连续 2 次失败触发兜底 |
| Skill 累积 token 膨胀 | 2-3 | 长对话时 disclosed_skills 可能过长；考虑截断或总结 |
| replay/eval 适配超期 | 4 | 阶段 1 就开始评估依赖；预留 buffer |
| 旧代码删除残留 | 4 | 分步标记 deprecated → 删除；`cargo check` 确认无引用 |
| Free tier budget 成本 | 1 | budget=2 意味着 Free 用户每次请求至少 2 次 LLM call；监控成本 |
