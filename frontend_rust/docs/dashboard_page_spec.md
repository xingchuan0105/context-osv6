# Dashboard Page Spec

## 页面定位
- 页面职责是知识库总览，不承载问答与编辑，只负责浏览、筛选、排序、进入具体 workspace。
- 用户在这里的核心动作只有四个：看列表、切视图、排序、创建新 notebook。

## 不可变视觉合同
- 整体是纯白背景，顶部是一条轻边框白色导航条。
- 品牌区在左上，只保留产品级品牌标识 `NotebookLM`，不在 notebook 行内再加图标。
- 主操作在右上：搜索、视图切换、排序、新建；其中 `新建` 是唯一深色实底按钮。
- 列表区是弱分割线表格，不使用卡片阴影、彩色标签或强背景块。
- 字体层级保持克制：页面标题最大，列表标题中等，元信息浅灰。

## 不可变布局合同
- 页面宽度居中，最大内容宽度约 1280px。
- 顶部第一层是产品头部；第二层是 tabs 和控制条；第三层是页面标题；第四层是 notebook 列表或卡片。
- tabs 只有 `全部` / `我的笔记本` 两个。
- 默认优先展示列表视图；卡片视图只是次级浏览方式。
- 列表列顺序固定：标题 / 来源 / 创建日期 / 角色 / 更多操作。

## 业务展示合同
- notebook 行内只展示真实字段：标题、来源数、日期、角色。
- 不从 notebook 名称推断图标、类别或颜色语义。
- 搜索按钮可以跳转搜索入口，但不在本页内塞复杂筛选逻辑。
- 排序只保留 `最近` 与 `标题` 两种。
- 新建 notebook 使用轻量创建弹层，创建成功后进入对应 workspace。
